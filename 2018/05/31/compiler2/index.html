2018-05-31   技术  编译原理 

第二章 词法分析

正则表达式(Regular Expression)

表示法

  • 符号 (symbol)
  • 可选 (alternation) a|b a或者b
  • 联结 (concatenation) a‧b ab
  • ϵ (epsilon) 空字符串
  • 重复 (repetition) M的克林闭包 M*， 表示M的0至多次联结运算
  • 扩展符号 [ ] 和 ? 和 +

正则表达式示例

正则表达式 token
if IF
[a-z][a-z0-9]* ID
[0-9]+ NUM
([0-9]+”.”[0-9]* ) \ ([0-9]*“.”[0-9]+) REAL
(“–”[a-z]*“\n”)\ (“\n”\ “\t”\ “ “)+ /*do nothing*/
. error()

语法规范应当是完整的，对于一个输入的字符串，我们总能找到一条匹配的语义规范

二义性处理

例如 if8 这样的字符串，可能被当成IF也可以被当成 ID 处理，怎么消除这样的歧义称作二义性处理：

  • 最长匹配 取可与任意一个规则匹配的最长字符串作为下一个单词
  • 规则优先 正则表达式的书写顺序是有意义的，对于特定的最长字串，第一个匹配的正则表达式决定了其token类型

有限自动机(Deterministic Finite Automation)

词法单词的有限自动机，圆圈表示状态，双圆圈表示终态

合并后的词法单词有限自动机

非确定有限自动机(Nondeterministic Finite Automation)

为什么需要非确定有限自动机？
非确定有限自动机是正则表达式和有限自动机之间的一种概念。由正则表达式很容易抓换成NFA。

正则表达式-> NFA

转换算法：把正则表达式转换为有一个头(结束状态)一个尾巴(初始边)的NFA。常用的是Thompson构造法

几种正则表达式的转换：

NFA-> DFA

有些时候，计算机可以直接使用NFA,不过常用的还是转换成DFA后再使用。
利用closure 和 DFAedge可以写出将NFA转换成DFA的形式算法，具体过程不赘述

Lex

词法分析的工作是将正则表达式转化成有限状态自动机，这个工作可以由计算机自动完成，常用的工具是Lex
